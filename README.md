# 🚌 公交路线优化沙盘

一个基于Canvas的公交路线优化模拟游戏，玩家需要在地图上规划公交站点和路线，平衡覆盖率与运营成本。

## ✨ 功能特性

- 🗺️ **地图网格系统** - 40x40网格的Canvas地图
- 🏠 **建筑放置** - 居民区（产生通勤需求）和商业区（目的地）
- 🚏 **站点规划** - 灵活放置公交站点
- 🛣️ **路线绘制** - 连接站点形成公交线路
- 🚶 **通勤模拟** - 真实的人员流动和公交运行模拟
- 📊 **实时指标** - 等待时间、满载率、覆盖率、运营成本
- 💾 **方案存储** - 支持保存和加载多个方案（后端+本地存储双备份）

## 🎮 游戏玩法

### 1. 放置建筑
- 选择"🏠 居民区"工具，点击地图放置居民区（会随机产生20-70人口）
- 选择"🏢 商业区"工具，点击地图放置商业区（作为通勤目的地）

### 2. 规划站点和路线
- 选择"🚏 公交站"工具，在合适位置放置公交站
- 选择"🛣️ 画线路"工具，依次点击两个公交站连接成线路
- 每条线路会自动发车，公交车在站点间往返运行

### 3. 运行模拟
- 点击"▶️ 开始模拟"启动通勤模拟
- 调整速度滑块（1-5x）控制模拟速度
- 观察实时数据面板，优化路线设计

### 4. 优化目标
- **覆盖率**：尽可能让更多居民区在公交站150范围内
- **等待时间**：增加发车频率减少乘客等待
- **满载率**：让公交保持较高但不过载的载客量
- **运营成本**：平衡站点数量和路线长度

## 🚀 安装和运行

### 方法1：直接打开（无需后端）
直接在浏览器中打开 `index.html` 即可游玩，方案会保存到浏览器本地存储。

### 方法2：启动后端服务器（推荐）

1. 安装依赖：
```bash
npm install
```

2. 启动服务器：
```bash
npm start
```

3. 在浏览器中访问：
```
http://localhost:3000
```

## 🔧 配置参数

可在 `game.js` 中调整以下参数：

- `GRID_SIZE` - 网格大小（默认40）
- `COSTS.stop` - 单个站点成本（默认50）
- `COSTS.route` - 路线每段成本（默认10）
- `BUS_CONFIG.capacity` - 公交车容量（默认30）
- `BUS_CONFIG.speed` - 公交车速度（默认2）
- `BUS_CONFIG.spawnInterval` - 发车间隔（默认300帧）

## 📊 评估指标说明

| 指标 | 说明 | 优化方向 |
|------|------|----------|
| 等待时间 | 乘客从到达站点到上车的平均时间 | ↓ 越低越好 |
| 满载率 | 公交车最高载客比例 | → 60-80%最佳 |
| 覆盖率 | 150m范围内有公交站的居民区比例 | ↑ 越高越好 |
| 运营成本 | 站点+路线+车辆的总成本 | ↓ 越低越好 |
| 乘客数 | 成功送达的总乘客数 | ↑ 越高越好 |

## 🎯 设计挑战

1. 尝试设计一条覆盖所有居民区但成本最低的路线
2. 挑战满载率保持在70%以上，同时等待时间低于50秒
3. 对比直线型路线和环形路线的效率差异
